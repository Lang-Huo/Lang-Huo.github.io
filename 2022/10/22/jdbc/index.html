JDBC总结

学习一个技术，首先要知道这个技术是做什么的，能够解决什么问题。

让我们来一步一步学习JDBC吧！！！

1.JDBC基本概念

JDBC： Java Data Base Connectivity， Java数据库连接

程序员，使用java去操作数据库，但是市面上的关系型数据库很多，比如(MySQL,Oracle…),每种关系型数据库都有自己的规范，想使用一套java代码操作所有的关系型数据库是行不通的。

所以期望用一套java代码，去操作所有的关系型数据库。因此Sun公司就定义了一套规则，起名叫JDBC,它定义了操作所有关系型数据的所有规范(接口)。

各大数据库厂商想要搭上java这个语言大佬，都必须遵守这一套接口，具体的实现类由各大厂商自己实现。（关系型数据实现类–数据库驱动）

⚠为了使自己的代码耦合性更小，尽量少使用使用实现类中的方法

JDBC的本质：

其实是官方（sun 公司）定义了一套操作所有关系型数据库的规则，即接口。各个数据库 厂商去实现这套接口（JDBC），提供数据库驱动 jar 包。我们可以使用这套接口编程，真正执行的代码是驱动 jar 包中的实现类，对应的实现方法

2.JDBC操作流程

  1. 导入驱动 jar 包
  2. 注册驱动 （让程序知道使用的是哪一个驱动包，哪个版本）
  3. 获取数库连接对象 Connection （这个对象就是本地的 java 代码和数据裤之间的桥梁对象）
  4. 定义 sql 语句（通过一些方式将 sql 语句发送给数据库，从而达到 java 代码操作数据库的要求）
  5. 获取执行 sql 语句的对象 Statement
  6. 执行 sql ,接收返回结果
  7. 处理结果
  8. 释放资源
